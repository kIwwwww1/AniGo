# Как клонировать приватный репозиторий GitHub

GitHub больше не поддерживает пароли для Git операций. Нужно использовать Personal Access Token.

---

## Решение: Создайте Personal Access Token

### Шаг 1: Создайте токен на GitHub

1. Откройте GitHub: https://github.com
2. Войдите в аккаунт `kIwwwww1`
3. Нажмите на ваш аватар (правый верхний угол) → **Settings**
4. В левом меню найдите **Developer settings**
5. Выберите **Personal access tokens** → **Tokens (classic)**
6. Нажмите **Generate new token** → **Generate new token (classic)**
7. Заполните форму:
   - **Note:** `AniGo VPS` (любое название)
   - **Expiration:** Выберите срок (например, 90 дней или No expiration)
   - **Select scopes:** Отметьте:
     - ✅ `repo` (полный доступ к репозиториям)
     - ✅ `workflow` (если используете GitHub Actions)
8. Нажмите **Generate token** внизу
9. **ВАЖНО:** Скопируйте токен сразу! Он показывается только один раз!
   - Выглядит примерно так: `ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### Шаг 2: Используйте токен вместо пароля

На сервере выполните:

```bash
git clone https://github.com/kIwwwww1/AniGo.git
```

Когда попросит:
- **Username:** `kIwwwww1`
- **Password:** Вставьте токен (не пароль от GitHub!)

**Если получаете ошибку 403:**
1. Убедитесь что токен создан с правами `repo` (полный доступ)
2. Проверьте что токен скопирован полностью (без пробелов)
3. Попробуйте создать новый токен и использовать его
4. Или используйте альтернативный способ ниже

## Альтернативный способ: Использовать SSH ключ

### Шаг 1: Создайте SSH ключ на сервере

```bash
# Генерируем SSH ключ
ssh-keygen -t ed25519 -C "your_email@example.com"

# Нажимайте Enter на все вопросы (или задайте свой пароль)

# Показываем публичный ключ
cat ~/.ssh/id_ed25519.pub
```

Скопируйте весь вывод (начинается с `ssh-ed25519...`)

### Шаг 2: Добавьте ключ в GitHub

1. Откройте GitHub: https://github.com/settings/keys
2. Нажмите **New SSH key**
3. **Title:** `VPS Server` (любое название)
4. **Key:** Вставьте скопированный ключ
5. Нажмите **Add SSH key**

### Шаг 3: Клонируйте через SSH

```bash
# Клонируем через SSH (не HTTPS!)
git clone git@github.com:kIwwwww1/AniGo.git
```

Теперь не нужно вводить пароль!

## Самый простой способ: Загрузить файлы напрямую

**Если получаете ошибку 403 или не хотите возиться с токенами:**

### Вариант 1: Скачать ZIP архив (РЕКОМЕНДУЕТСЯ)

1. Откройте репозиторий: https://github.com/kIwwwww1/AniGo
2. Нажмите зеленую кнопку **Code** → **Download ZIP**
3. Распакуйте архив на вашем компьютере
4. Загрузите файлы на сервер:

**Через FileZilla/WinSCP:**
- Хост: `77.222.58.178`
- Порт: `22`
- Протокол: `SFTP`
- Логин: `root`
- Пароль: (если настроили, или используйте VNC консоль)
- Загрузите папку `AniGo` в `/opt/`

**Или через SCP (на вашем компьютере):**
```bash
# На вашем компьютере (Mac/Linux)
scp -r /path/to/AniGo root@77.222.58.178:/opt/

# На Windows (PowerShell)
scp -r C:\path\to\AniGo root@77.222.58.178:/opt/
```

**После загрузки на сервере:**
```bash
cd /opt
mv AniGo anigo  # Переименовываем для удобства
cd anigo
```

### Вариант 2: Сделать репозиторий публичным временно

1. Откройте: https://github.com/kIwwwww1/AniGo/settings
2. Прокрутите вниз до **Danger Zone**
3. Нажмите **Change visibility** → **Make public**
4. Клонируйте репозиторий (теперь без токена):
```bash
git clone https://github.com/kIwwwww1/AniGo.git
```
5. После клонирования можете вернуть приватность

### Вариант 2: Через Portainer (загрузка файлов)

1. В Portainer создайте volume: `anigo-data`
2. Загрузите файлы через веб-интерфейс или используйте контейнер для клонирования

2. Используйте его как пароль при клонировании

## Если забыли токен

Просто создайте новый:
1. GitHub → Settings → Developer settings → Personal access tokens
2. Удалите старый (если нужно)
3. Создайте новый
4. Используйте новый токен
