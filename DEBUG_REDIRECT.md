# Отладка редиректа на portainer.io

## Проблема
Браузер перенаправляет на https://www.portainer.io/ вместо вашего сайта.

## Возможные причины

### 1. Проблема с DNS / hosts файлом
Проверьте файл `/etc/hosts`:
```bash
cat /etc/hosts | grep -i "yumivo\|portainer"
```

Если там есть записи для portainer, удалите их.

### 2. Кэш браузера
Очистите кэш браузера:
- **Chrome/Safari**: Cmd+Shift+Delete
- Или откройте сайт в режиме инкогнито/приватном режиме

### 3. Расширения браузера
Отключите все расширения браузера, особенно:
- VPN расширения
- Блокировщики рекламы
- Расширения безопасности

### 4. Проверка через curl
Проверьте, что возвращает сервер:
```bash
curl -k -I https://localhost
curl -k https://localhost | head -20
```

### 5. Проверка через другой браузер
Попробуйте открыть сайт в другом браузере (если используете Chrome, попробуйте Safari или Firefox).

### 6. Проверка DNS
Если используете домен yumivo.ru:
```bash
dig yumivo.ru +short
nslookup yumivo.ru
```

## Решение

### Вариант 1: Использовать localhost напрямую
Откройте в браузере:
```
http://localhost
```

### Вариант 2: Очистить кэш DNS
```bash
# macOS
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder
```

### Вариант 3: Проверить настройки прокси
Если используете прокси или VPN, отключите их временно.

## Проверка работы сайта

Ваш сайт должен возвращать HTML с заголовком "Yumivo — смотри аниме онлайн".

Проверьте через curl:
```bash
curl -k https://localhost | grep -i "yumivo\|title"
```

Если видите "Yumivo", значит сайт работает правильно, проблема в браузере или DNS.
